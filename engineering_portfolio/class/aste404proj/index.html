2D Mapper

Goal

Construct a prototype for a vehicle that can map out lunar caves.

Cause

A class in Fall 2022 required us to form groups and come up with a project that utilizes numerical methods.

Process

First, we had to find the sensors needed for this project. One of my groupmates had a Raspberry Pi, so we bought an IMU (Inertial Measurement Unit) and a rotating LiDAR. The former was used to determine the rover's state while the latter was used to find the distance from the rover to the walls. We split the work such that the four of us would take on one different part of the task: IMU integration, LiDAR integration, sensor fusion, and data analysis. I was in charge of sensor fusion, but while I was waiting for the IMU and LiDAR code, I wrote a set of Python functions that allowed for the rotating of vectors using quaternions. Those functions allowed us to keep the gravity vector pointing down even when the IMU was at an angle via the following steps:

  1. Obtain the roll-pitch-yaw (RPY) set from the IMU by integrating the angular velocities.
  2. Call one of the functions to convert the RPY set into a quaternion.
  3. Call another function to rotate the downward gravity vector using that quaternion (this function calls another function that does the quaternion multiplication).
