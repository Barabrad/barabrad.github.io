Space Domain Awareness (Summer 2023)

Goal

Add functionality to existing analyses so that they can take state vectors and covariances as inputs.

Cause

The analyses in question could take in only TLEs (Two-Line Element Sets) as inputs. State vectors and covariances allow for the propagation of a satellite to be more precise while preserving and propagating the uncertainty in its state.

Process

Before I could work on extending the valid inputs, I had to get the code running on my laptop. Because the code was written on Windows computers, the MATLAB scripts and wrappers (for the dynamic libraries) would not run properly on my Mac. I could not connect to the recommended remote server either, so I started working on making the repository Mac-friendly while waiting for a Windows laptop. With some help from my seniors, I was able to get the code running properly on my Mac before the replacement laptop arrived. (I documented the steps I took in case someone else in the future has the same issue.)

Once I got the code working, I began updating the code to allow the new inputs. However, I had to translate some Python code from a sister repository to deal with the covariances. To test the code, I wrote MATLAB scripts that could convert TLEs to state vectors (these conversion scripts and their outputs were solely meant for testing the code, not for actual analyses).

Results

I was able to extend an analysis script to include state vectors and covarainces before my internship ended. The tests I ran with the converted files showed that the code was able to parse the new inputs correctly (the initial states of all inputs were close enough, due to rounding errors).
